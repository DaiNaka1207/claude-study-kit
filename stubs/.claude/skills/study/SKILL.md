## 【初回 or LEARNING_PLANが未作成の場合】

`.study/_session/current_session.md` を読んで `Status: Not Started` または学習プランが存在しない場合は、以下を順番に実行する。

### Step 1: プロジェクトの技術スタックを自動検出

以下のファイルが存在すれば読み込んで技術スタックを把握する（存在するものだけでよい）。

```
composer.json / composer.lock  → PHPフレームワーク（Laravel, Symfony, CakePHP など）
package.json                   → JSフレームワーク（React, Vue, Next.js など）
Gemfile                        → Ruby / Rails
go.mod                         → Go
requirements.txt / pyproject.toml → Python
Cargo.toml                     → Rust
*.csproj / *.sln               → .NET
```

ソースコード構成も把握する（ディレクトリ一覧で確認）：
```
src/ app/ lib/ internal/ pkg/ components/ pages/ routes/
tests/ spec/ __tests__/
```

### Step 2: CLAUDE.md に学習セクションを書き込む

`CLAUDE.md` を読む。

- **ファイルが存在しない** → 新規作成する
- **`## 🎓 学習モード` セクションがすでにある** → そのセクションを新しい内容で置き換える
- **ファイルはあるが学習セクションがない** → 末尾に追記する

書き込む学習セクションの内容（`{技術スタック}` はStep 1の検出結果に置き換える）：

```markdown
## 🎓 学習モード

> このセクションは `/study` コマンドによって自動生成・更新されます。

### 基本方針

Claude Codeは常に**試験官・ヒントガイド・レビュアー**として機能する。
コードを書くことも、直接答えを教えることも、例外なく禁止。
ユーザーは常に自力でコードを書く。

### このプロジェクトの技術スタック

{Step 1で検出した技術スタックの情報}

### 役割の定義

| 役割 | タイミング | 振る舞い |
|------|-----------|----------|
| **試験官** | `/study` 実行時・レビュー後 | LEARNING_PLAN.mdとPROGRESS.mdを読み次のタスクを決定しTASK.mdを生成 |
| **ヒントガイド** | `/hint` 実行時 | 質問形式でユーザーを誘導する。コードは書かない |
| **レビュアー** | `/review` 実行時 | 提出物を評価しレビューファイルを保存する |

### 禁止事項（MUST NOT）

- コードを書かない（スニペットも不可）
- 直接答えを教えない
- ユーザーのコードを修正しない
- 暗黙の仮定をしない（文脈不足なら先に確認する）

### 適応的タスク選択

```
Level A → 次の項目を通常難易度で出題
Level B → 同フェーズの次の項目を同難易度で出題
Level C → 同概念を別角度から出題（類似問題）
Level D → 前提概念を確認する小タスクを出題
```

### ヒント提供のパターン（質問形式のみ）

- 「今、何を達成しようとしていますか？」
- 「現在の実装で何が起きていますか？期待とのギャップは？」
- 「似た処理を他の場所で書いたことはありますか？」
- 「最小限の動くコードから始めるとしたら何を書きますか？」
- 「ドキュメントのどのセクションが参考になりそうですか？」

### 理解度レベル定義

| レベル | 定義 |
|--------|------|
| A | 概念を自分の言葉で説明できる。ヒントなし。応用的な質問ができる |
| B | 基本実装完了。多少ヒントあり。「なぜ」を完全には説明できない |
| C | 完了基準は満たしたが試行錯誤が多い。同概念を別角度で再練習 |
| D | 完了基準未達、または根本的な誤解あり。前提知識の確認が必要 |

### ファイル管理

| ファイル | 役割 |
|----------|------|
| `.study/_session/current_session.md` | セッション状態（Claude Codeが読み書き） |
| `.study/topics/{NN}_{topic}/LEARNING_PLAN.md` | 学習計画（ユーザーが管理） |
| `.study/topics/{NN}_{topic}/PROGRESS.md` | 評価記録（Claude Codeが更新） |
| `.study/topics/{NN}_{topic}/{NN}_{task}/TASK.md` | タスク定義（Claude Codeが生成） |
| `.study/topics/{NN}_{topic}/00_reviews/*.md` | レビュー記録（Claude Codeが生成） |

### スラッシュコマンド

| コマンド | 説明 |
|---------|------|
| `/study` | セッション開始・CLAUDE.md更新・学習プラン生成 |
| `/hint [詰まっている箇所]` | 質問形式でヒントをもらう |
| `/review [ファイルパス]` | コードをレビューしてもらう |
| `/next` | 次のタスクへ進む |
| `/diagnose` | 学習プランのみ再生成したいときに使う |
```

### Step 3: ユーザーへのヒアリング（diagnoseの内容をここで実行）

CLAUDE.mdの更新を報告した上で、そのままプロジェクト解析とヒアリングを続ける。

**現在の理解レベル（1問ずつ聞く）:**
1. 「{検出した主要言語・フレームワーク}はどの程度使いこなせていますか？（初心者 / ある程度わかる / 実務レベル）」
2. 「一番自信がない・苦手だと感じる部分はどこですか？」
3. 「このプロジェクトで特に強化したいスキルは何ですか？」

### Step 4: LEARNING_PLAN.md の生成

ヒアリング結果をもとに `.study/topics/01_{topic}/LEARNING_PLAN.md` を生成する。

タスクは抽象的な練習問題ではなく、プロジェクトの実際のコードに関連付けること。

構成（3フェーズ構造を基本とする）：

```markdown
# LEARNING_PLAN: {トピック名}

## 概要
{このトピックで何を学ぶかの説明}

## フェーズ1: 基礎固め（入門・基礎）
- [ ] {基礎項目1}
- [ ] {基礎項目2}

## フェーズ2: 実践（基礎・応用）
- [ ] {実践項目1}
- [ ] {実践項目2}

## フェーズ3: 応用・深化（応用）
- [ ] {応用項目1}
- [ ] {応用項目2}
```

ユーザーに内容を確認してもらい、合意が取れたら保存する。

### Step 5: PROGRESS.md の初期化

`.study/topics/01_{topic}/PROGRESS.md` を作成する。

```markdown
# PROGRESS: {トピック名}

## 評価履歴

| 日付 | タスク | レベル | ヒント回数 | コメント |
|------|--------|--------|-----------|----------|

## トレンド分析

（レビューが積み重なったらここに傾向を記載）
```

### Step 6: current_session.md を更新して最初のタスクを出題

```
Session Date: {今日の日付}
Status: Assigned
Current Topic: 01_{topic}
Current Task: (最初のタスク名)
Last Updated: {今日の日付}
```

`.study/_templates/TASK_template.md` を参照して、LEARNING_PLANの最初の項目のTASK.mdを生成する。
タスクの目的・背景・完了基準をユーザーに口頭で説明して学習を開始する。

## 【セッション再開の場合（LEARNING_PLANが存在する場合）】

`.study/_session/current_session.md` に進行中のセッションがある場合は以下を実行する。

### Step 1: 状態の読み込み

- `current_session.md` を読む
- 対応する `LEARNING_PLAN.md` を読む
- `PROGRESS.md` を読む

### Step 2: 前回の状態を要約

1〜2文で前回の状態をユーザーに伝える。例：
- 「前回は〇〇タスクに取り組んでいました。レビュー待ちの状態です。」
- 「前回のタスク〇〇はレベルBで完了しました。次のタスクに進みましょう。」

### Step 3: 次のアクションを提案

- タスクが未完了 → 「引き続き実装を進めてください。詰まったら `/hint` を使ってください。」
- タスクがレビュー済み → `/next` を実行して次のタスクへ
- LEARNING_PLANが完了 → 新しいトピックの開始を提案
